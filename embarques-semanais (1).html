📋 Como configurar o banco de dados (Firebase)

Passo 1 — Criar conta no Firebase (grátis)
Acesse console.firebase.google.com e faça login com sua conta Google.

Passo 2 — Criar um projeto
Clique em "Adicionar projeto", dê o nome que quiser (ex: super-safra), desative o Google Analytics se não precisar, e clique em "Criar projeto".

Passo 3 — Adicionar app Web
Dentro do projeto, clique no ícone </> (Web), dê um apelido (ex: embarques) e clique em "Registrar app".
Copie o objeto firebaseConfig que aparece na tela.

Passo 4 — Colar as credenciais neste arquivo
Abra embarques-semanais.html em qualquer editor de texto (Bloco de Notas serve), localize o trecho:
const firebaseConfig = { apiKey: "COLE_AQUI" …
e substitua cada campo pelo valor do seu projeto.

Passo 5 — Criar o banco Firestore
No painel do Firebase, vá em Firestore Database → Criar banco de dados.
Escolha o modo Produção e selecione a região southamerica-east1 (São Paulo).

Passo 6 — Liberar acesso de leitura/escrita
Em Firestore → Regras, cole as regras abaixo e publique:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /embarques/{doc} {
      allow read, write: if true;
    }
  }
}
(Isso libera acesso público. Para uso interno restrito, configure autenticação depois.)

Passo 7 — Publicar no GitHub Pages
1. Crie uma conta no GitHub se não tiver.
2. Crie um repositório público (ex: embarques-super-safra).
3. Faça upload do embarques-semanais.html com as credenciais já preenchidas.
4. Vá em Settings → Pages, selecione a branch main e salve.
5. Em alguns minutos seu sistema estará em:
https://SEU-USUARIO.github.io/embarques-super-safra/embarques-semanais.html

✅ Plano gratuito do Firebase suporta 50.000 leituras e 20.000 escritas por dia — mais do que suficiente para uso interno de uma equipe pequena.
